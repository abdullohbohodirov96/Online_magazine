# BozorMarket — MVP Онлайн-Магазина Продуктов (Маркетплейс)

Это MVP онлайн-магазина продуктов питания в стиле маркетплейса для одного магазина, созданный на **Next.js** и **Prisma** с базой данных **PostgreSQL**.

Интерфейс полностью на русском языке, валюта — сум (UZS).

---

## 🚀 Основные возможности

### 1. Для Покупателя
- **Главная страница**: красивый баннер, сетка категорий, популярные и скидочные товары.
- **Интерактивный каталог**: поиск товаров в реальном времени, фильтрация по категориям без перезагрузки.
- **Карточка товара**: отображение остатков, статусы («В наличии», «Мало осталось», «Нет в наличии»), добавление в корзину.
- **Детальный просмотр**: модальное окно с подробным описанием товара, штрихкодом, кодом номенклатуры и единицами измерения.
- **Интерактивная корзина**: управление количеством товаров с подсчетом общей суммы. Корзина работает локально (для гостей) и автоматически синхронизируется с БД при авторизации.
- **Оформление заказа**: форма ввода имени, телефона, адреса доставки и комментария курьеру.
- **Авторизация по OTP**: регистрация и вход по номеру телефона через 4-значный SMS-код (в dev-режиме код выводится в консоль терминала).
- **Личный кабинет**: история заказов с деталями (список товаров, адрес доставки, статус заказа).

### 2. Для Администратора
- **Панель управления**: доступна только пользователям с ролью `ADMIN`.
- **Управление заказами**: просмотр всех заказов и изменение их статуса (Новый, Принят, Собирается, В доставке, Завершен, Отменен).
- **Управление товарами (CRUD)**: добавление, редактирование, активация/деактивация и удаление товаров. Поддерживает ручной режим создания и интеграционный.
- **Интеграция с учетной системой**:
  - Настройка параметров интеграции (Enabled/Disabled, API URL, Ключ, интервалы).
  - Поиск и автозаполнение товаров из внешней базы (`ExternalProduct`) по штрихкоду/коду/имени.
  - Синхронизация цен и остатков с тонкими правилами (`syncPrice` и `syncStock` для каждого товара).
  - Просмотр логов синхронизации (`IntegrationLog`).

### 3. API Синхронизации (Интеграция)
- **Эндпоинт**: `POST /api/integration/products-sync`
- **Безопасность**: проверка заголовка `x-integration-key`.
- **Логика работы**:
  - Сопоставление по `barcode` или `nomenclatureCode`.
  - Автоматическое создание категорий по имени, если они не существовали.
  - Логирование процесса в `IntegrationLog`.

## 🛠 Технологический стек
- **Frontend**: Next.js 16 (App Router, React 19, Client & Server Components).
- **Styling**: Vanilla CSS (с CSS переменными, плавными переходами и премиальным UI).
- **ORM**: Prisma 7 (с поддержкой Driver Adapters).
- **Database**: PostgreSQL (библиотека `pg` и `@prisma/adapter-pg`).
- **Auth**: JWT-сессия (`jose`) в HttpOnly Cookies + OTP (SMS mock / Eskiz API).

## ⚙️ Установка и запуск проекта в dev-режиме

### 1. Установка зависимостей
```bash
npm install
```

### 2. Настройка `.env`
Создайте файл `.env` на основе `.env.example` и укажите данные подключения к локальной БД PostgreSQL:
```env
DATABASE_URL="postgresql://username:password@localhost:5432/grocery_db?schema=public"
JWT_SECRET="super-secret-jwt-key-grocery-marketplace-2026"
INTEGRATION_SECRET_KEY="grocery-integration-key-xyz-123"
SMS_PROVIDER="mock"
SMS_PROVIDER_API_KEY="mock-sms-provider-key-abc-456"
NEXT_PUBLIC_APP_URL="http://localhost:3000"
```

### 3. Запуск миграций и seed-скрипта
```bash
# Применить миграции
npx prisma migrate dev --name init

# Запустить seed
npm run db:seed
```

### 4. Запуск локально
```bash
npm run dev
```

## 🌐 Production Deployment

Данный проект полностью готов к развертыванию в продакшн с базой данных PostgreSQL (например, на Neon или Supabase) и хостингом на Vercel.

### А) Создание базы данных PostgreSQL
1. **Neon**:
   - Перейдите на [neon.tech](https://neon.tech/) и создайте новый проект.
   - Скопируйте строку подключения `DATABASE_URL` (она должна выглядеть как `postgresql://user:pass@ep-hostname.us-east-2.aws.neon.tech/neondb?sslmode=require`).
2. **Supabase**:
   - Зарегистрируйтесь на [supabase.com](https://supabase.com/) и создайте новый проект.
   - Перейдите в раздел **Project Settings -> Database**.
   - Скопируйте строку подключения `DATABASE_URL`. Для serverless-окружения (Vercel) рекомендуется использовать Connection Pooler (порт `6543`, режим `Transaction`).

### Б) Добавление переменных окружения (Environment Variables) на Vercel
При создании проекта в Vercel добавьте следующие переменные окружения:
- `DATABASE_URL` — строка подключения к Neon или Supabase.
- `JWT_SECRET` — длинная случайная строка для защиты сессий.
- `INTEGRATION_SECRET_KEY` — ключ для синхронизации товаров из внешней учетной программы.
- `SMS_PROVIDER` — `eskiz` для отправки через реального провайдера или `mock` для логирования кодов в консоль Vercel.
- `SMS_PROVIDER_API_KEY` — API-ключ SMS-сервиса (необязательно для mock).
- `NEXT_PUBLIC_APP_URL` — адрес вашего сайта на Vercel (например, `https://your-app.vercel.app`).
- **Для интеграции с Eskiz (если SMS_PROVIDER=eskiz)**:
  - `ESKIZ_EMAIL` — ваш email на Eskiz.uz.
  - `ESKIZ_PASSWORD` — ваш пароль на Eskiz.uz.
  - `ESKIZ_FROM` — ваш одобренный никнейм/альфа-имя (по умолчанию `4546`).

*Примечание: Vercel автоматически запустит скрипт `postinstall` для генерации Prisma Client при сборке.*

### В) Запуск миграций в продакшне
Выполните следующую команду на локальном компьютере, указав продакшн-строку подключения:
```bash
DATABASE_URL="ваша_production_db_url" npx prisma migrate deploy
```
*Команда применит все SQL-миграции из папки `prisma/migrations` на боевую базу данных.*

### Г) Наполнение базы данных (Seed)
Для добавления администратора и демо-товаров в боевую базу данных выполните:
```bash
DATABASE_URL="ваша_production_db_url" npm run db:seed
```

### Д) Проверка администратора
- **Номер телефона администратора**: `+998917850090`
- **Роль**: `ADMIN`
- **Код авторизации**:
  - В режиме `SMS_PROVIDER=mock`: код отобразится в логах Vercel (раздел **Logs** в дашборде проекта Vercel).
  - В режиме `SMS_PROVIDER=eskiz`: код придет на телефон в реальном SMS.
